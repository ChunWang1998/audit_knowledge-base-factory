The main reference issue title in knowledgeBase: State desynchronization between contracts (cross-contract accounting inconsistencies)
File Location: contracts/src/yield/LidoStVaultYieldProvider.sol (reportYield, _payMaximumPossibleLSTLiability, _syncExternalLiabilitySettlement) + delegatecall path from YieldManager.sol
Issue Description: External Lido actions (VAULT_HUB.settleLidoFees, dashboard.disburseFee) can reduce liabilityShares between report cycles; internal lstLiabilityPrincipal sync assumes external settlement is always allocated interest-first, but RoundUp vs floor rounding in getPooledEthBySharesRoundUp / getSharesByPooledEth creates desync with userFunds and totalValue.
Attack scenario: Attacker triggers external settlement + times a reportYield (or permissionless unstake) to exploit the desynced obligation accounting, causing over/under-reported yield → one user steals others’ share of rewards or inflates/deflates reported userFunds, leading to direct theft of user funds (in-scope Critical) or permanent freezing. Verified by code: lstLiabilityPrincipalSynced = STETH.getPooledEthBySharesRoundUp(_liabilityShares) vs earlier min/floor logic. Strictly in-scope as it affects on-chain accounting state.
